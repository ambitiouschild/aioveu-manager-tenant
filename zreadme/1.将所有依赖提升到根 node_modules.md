shamefully-hoist=true的意思：

在 pnpm 中，默认情况下依赖是符号链接的，不是平铺安装。这意味着：

•你的项目只能直接访问 package.json中声明的依赖

•依赖的依赖（子依赖）有自己的独立目录

•第三方组件如果不在你的 package.json中，可能会访问不到

shamefully-hoist=true会：

1.将所有依赖提升到根 node_modules

2.让 Vite/Webpack 能直接找到它们

3.类似 npm/yarn 的默认行为

🔧 在你的项目中操作
1. 查看当前状态
# 查看当前 .npmrc
cat .npmrc

# 查看 pnpm 配置
pnpm config list
2. 修改配置
在项目根目录创建/修改 .npmrc文件：

# .npmrc
shamefully-hoist=true
# 可选：设置 store 目录
store-dir=node_modules/.pnpm-store
3. 重新安装依赖
# 删除现有 node_modules
rm -rf node_modules
# 重新安装
pnpm install
📁 安装后的目录结构
开启 shamefully-hoist=true后，结构变为：



node_modules/
├── uni-tree-view/                    # ✅ 现在可以直接访问
│   ├── src/
│   │   └── components/
│   │       └── uni-tree-view/
│   │           └── uni-tree-view.vue
│   └── package.json
├── vue/                              # Vue 也在根目录
├── uni-ui/                           # 其他依赖都在根目录
└── .pnpm/                            # pnpm 的内部链接
💡 分包问题
你的项目结构是分包架构：



src/packageC/                         # 分包C
└── pages/aioveuMallPms/
    └── CategoryTree/
        └── components/
            └── CategoryTree.vue      # 你的组件
问题：分包中的组件无法直接访问根 node_modules中的依赖。

🎯 分包专用解决方案
方案1：在分包中安装依赖
bash
复制
# 进入分包目录
cd src/packageC

# 在分包中安装 uni-tree-view
pnpm add uni-tree-view

# 或者从根目录安装到分包
pnpm add uni-tree-view --filter packageC
方案2：修改分包配置
在 packageC的 package.json中添加依赖：



// src/packageC/package.json
{
  "name": "package-c",
  "dependencies": {
    "uni-tree-view": "^1.0.0"
  }
}
方案3：使用别名（推荐）
在 vite.config.ts中配置别名：

typescript

// vite.config.ts
import { defineConfig } from 'vite'
import uni from '@dcloudio/vite-plugin-uni'
import path from 'path'

export default defineConfig({
  plugins: [uni()],
  resolve: {
    alias: {
      // 为分包配置别名
      'uni-tree-view/components/uni-tree-view.vue': 
        path.resolve(__dirname, 'node_modules/uni-tree-view/src/components/uni-tree-view/uni-tree-view.vue')
    }
  }
})

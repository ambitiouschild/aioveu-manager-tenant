## ❌ 报错根因（一句话）

> `processedConfig` 是在 `#ifdef H5` 里定义的  
> 但在非 H5 平台（App / 小程序）里根本不存在

所以这里必炸：

```ts
uni.request({
  ...options,
  url: `${baseApi}${processedConfig.url}`, // ❌ processedConfig is not defined
  header: headers,
});
```

## ✅ 终局正确写法（✅ 推荐）

### ✅ 把 `processedConfig` 放到 **条件编译外面**

```ts
export async function request2<T>(options: UniApp.RequestOptions): Promise<ResponseData<T>> {
  let baseApi = import.meta.env.VITE_APP_API_URL;

  // #ifdef H5
  baseApi = import.meta.env.VITE_APP_BASE_API;
  // #endif

  // ✅ 必须放这里（所有平台都有）
  const processedConfig = { ...options };

  try {
    Object.assign(processedConfig, await requestInterceptor(processedConfig));
  } catch (error) {
    return Promise.reject(error);
  }

  return new Promise((resolve, reject) => {
    const headers = {
      ...processedConfig.header,
    };

    if (processedConfig.url?.includes("/oauth2/token")) {
      headers["Content-Type"] = "application/x-www-form-urlencoded";
      delete headers["Bearer"];
    } else {
      const token = getToken();
      if (token) {
        headers["Authorization"] = `Bearer ${token}`;
      }
    }

    uni.request({
      ...processedConfig,
      url: `${baseApi}${processedConfig.url}`,
      header: headers,
      success: (response) => {
        const resData = response.data as ResponseData<T>;

        if (resData.code === ResultCodeEnum.SUCCESS) {
          resolve(resData);
        } else {
          reject({
            code: resData.code,
            message: resData.msg,
          });
        }
      },
      fail: reject,
    });
  });
}
```

| 点                            | 原因             |
| ----------------------------- | ---------------- |
| `processedConfig` 在外层      | ✅ 所有平台都存在 |
| 拦截器仍然只在 H5 需要        | ✅ 不影响         |
| `url / header` 统一来源       | ✅ 不再 undefined |
| uni.request 使用同一个 config | ✅ 安全           |

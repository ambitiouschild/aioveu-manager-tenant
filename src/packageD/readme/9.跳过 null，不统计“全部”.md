## ❌ 报错含义（直译）

```text
TS2538: Type 'null' cannot be used as an index type.
```

意思是：

> `status.value` 现在是 `null`
> 但你把它当成 对象索引 用了

---

## ❌ 出错位置（你这段代码）

```ts
orderStatusList.value.forEach((status) => {
  const count = statistics.statusCounts[status.value] || 0;
  //                                     ^^^^^^^^^^^^^^
  //                                     status.value 可能是 null ❌
  status.count = count;
});
```

现在你的 `orderStatusList` 里已经有这一项了：

```ts
{ value: null, label: "全部", class: "all", count: 0 }
```

👉 于是这里等价于：

```ts
statistics.statusCounts[null] // ❌ TypeScript 直接拒绝
```

---

## ✅ 正确修复方式（推荐 3 选 1）

---

## ✅✅✅ 方案一（最推荐）：跳过 `null`，不统计“全部”

### ✅ 逻辑上也是最对的

> “全部”本来就不应该有数量，数量是后端按状态算的

### ✅ 改法（最小、最干净）

```ts
const updateOrderStats = (statistics: any) => {
  if (statistics.statusCounts) {
    orderStatusList.value.forEach((status) => {
      // ✅ 跳过 null
      if (status.value === null) return;

      const count = statistics.statusCounts[status.value] ?? 0;
      status.count = count;
    });
  }

  todayOrders.value = statistics.todayOrderCount || 0;
  pendingOrders.value = statistics.pendingCount || 0;
  todayIncome.value = statistics.todayIncome || 0;
};
```

✅ 不报错  
✅ 语义正确  
✅ 不影响 UI  
✅ 不需要后端配合  

👉 这是企业级写法

## ✅ 顺便帮你把这段改成「类型安全版」（进阶）

```ts
interface StatusCount {
  [key: string]: number;
}

interface OrderStatistic {
  statusCounts?: StatusCount;
  todayOrderCount?: number;
  pendingCount?: number;
  todayIncome?: number;
}

const updateOrderStats = (statistics: OrderStatistic) => {
  if (statistics.statusCounts) {
    orderStatusList.value.forEach((status) => {
      if (status.value === null) return;
      status.count = statistics.statusCounts[status.value] ?? 0;
    });
  }

  todayOrders.value = statistics.todayOrderCount ?? 0;
  pendingOrders.value = statistics.pendingCount ?? 0;
  todayIncome.value = statistics.todayIncome ?? 0;
};
```

✅ 以后 再也不会出现 TS2538

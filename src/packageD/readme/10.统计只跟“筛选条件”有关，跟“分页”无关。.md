## ✅ 一句话结论（先给你）

> 订单统计接口只应该在「筛选条件变化」时调用一次，而不是每次分页加载都调用。

👉 你现在的代码：每翻一页都会重新调一次统计接口，这是重复获取的根源。

## 三、✅ 正确改造方案（最小改动，推荐）

### ✅ 1️⃣ 把「统计接口」从 `loadOrders` 里**彻底移除**

```ts
const loadOrders = async (reset = false) => {
  if (loading.value) return;
  if (reset) {
    pageNum.value = 1;
    hasMore.value = true;
    orderList.value = [];
  }
  if (!hasMore.value) return;

  loading.value = true;

  try {
    const params = getFilterParams();
    const response = await listOrdersWithPage(params);

    if (response.code === "00000") {
      const data = response.data;
      orderList.value = reset
        ? data.list
        : [...orderList.value, ...data.list];
      hasMore.value = orderList.value.length < data.total;
      pageNum.value++;
    }
  } catch (e) {
    uni.showToast({ title: "加载失败", icon: "error" });
  } finally {
    loading.value = false;
    refreshing.value = false;
  }
};
```

✅ 统计接口不在里面了

### ✅ 2️⃣ 单独写一个「加载统计的方法」

```ts
const loadOrderStatistics = async () => {
  try {
    const params = getFilterParams();
    const statistics = await getOrderStatistics(params);
    updateOrderStats(statistics);

    console.error("加载统计失败", e);
  }
};
```

这个错误是因为 TypeScript 找不到 `uni.onShareAppMessage`的类型定义。在 uni-app TypeScript 项目中，需要扩展 uni 的类型定义。以下是解决方案：

## 方案1：扩展 uni-app 类型定义（推荐）

**步骤1：创建类型声明文件**

```
// types/uni-app.d.ts
import { Uni } from '@dcloudio/types'

// 扩展 Uni 接口
declare module '@dcloudio/types' {
  interface Uni {
    // 分享给朋友
    onShareAppMessage(options: () => UniApp.ShareOptions): void
    
    // 分享到朋友圈
    onShareTimeline(options: () => UniApp.ShareTimelineOptions): void
    
    // 其他的 uni 方法类型定义...
  }
}

// 或者更简单的方式，直接扩展全局 Uni 接口
declare global {
  interface Uni {
    onShareAppMessage(options: () => UniApp.ShareOptions): void
    onShareTimeline(options: () => UniApp.ShareTimelineOptions): void
  }
  
  namespace UniApp {
    interface ShareOptions {
      title?: string
      path?: string
      imageUrl?: string
      desc?: string
      content?: string
      success?: (res: any) => void
      fail?: (err: any) => void
      complete?: () => void
    }
    
    interface ShareTimelineOptions {
      title?: string
      query?: string
      imageUrl?: string
      success?: (res: any) => void
      fail?: (err: any) => void
      complete?: () => void
    }
  }
}
```

**步骤2：在 tsconfig.json 中包含类型文件**

```
{
  "compilerOptions": {
    "types": [
      "@dcloudio/types",
      "miniprogram-api-typings"
    ],
    "typeRoots": [
      "./node_modules/@types",
      "./types"
    ]
  },
  "include": [
    "src/**/*.ts",
    "src/**/*.vue",
    "types/**/*.d.ts"
  ]
}
```

## 方案2：使用类型断言（快速修复）

在代码中使用类型断言绕过类型检查：

```
// composables/useShare.ts
import { ref, onMounted, onUnmounted } from 'vue'

// 定义分享选项类型
interface ShareOptions {
  title?: string
  path?: string
  imageUrl?: string
  query?: string
  success?: (res?: any) => void
  fail?: (err?: any) => void
}

export function useShare(defaultConfig: ShareOptions = {}) {
  // 默认分享配置
  const defaultShareConfig: ShareOptions = {
    title: '可我不敌心软~',
    path: '/pages/index/index',
    imageUrl: 'https://cdn.aioveu.com/aioveu-server/avatar/avatar.png',
    query: 'key=value'
  }
  
  // 合并配置
  const shareConfig = ref({
    ...defaultShareConfig,
    ...defaultConfig
  })
  
  // 设置分享的方法
  const setupShare = (customConfig: ShareOptions = {}) => {
    // 更新配置
    if (customConfig) {
      shareConfig.value = { ...shareConfig.value, ...customConfig }
    }
    
    // 使用类型断言
    const uniWithShare = uni as any
    
    // 设置分享给朋友
    uniWithShare.onShareAppMessage(() => ({
      title: shareConfig.value.title,
      path: shareConfig.value.path + (shareConfig.value.query ? `?${shareConfig.value.query}` : ''),
      imageUrl: shareConfig.value.imageUrl,
      success: (res?: any) => {
        console.log("分享成功", res)
      },
      fail: (err?: any) => {
        console.log("分享失败", err)
      }
    }))

    // 设置分享到朋友圈
    uniWithShare.onShareTimeline(() => ({
      title: shareConfig.value.title,
      query: shareConfig.value.query,
      imageUrl: shareConfig.value.imageUrl,
      success: (res?: any) => {
        console.log("分享到朋友圈成功", res)
      },
      fail: (err?: any) => {
        console.log("分享到朋友圈失败", err)
      }
    }))
  }
  
  return {
    shareConfig,
    setupShare
  }
}
```

## 方案4：创建类型安全的 Hook（推荐完整方案）

## 推荐方案

**对于长期项目，推荐使用方案1（扩展类型定义）**，原因：

1. 类型安全，代码提示完整
2. 可维护性好
3. 团队协作统一

**对于快速开发，可以使用方案2或5（类型断言）**，简单直接。

**如果你的 uni-app 版本支持 `definePageConfig`，方案3是官方推荐的方式。**

在 Vue 3 开发中，‌**composables**‌ 是一种基于 ‌**Composition API**‌ 的‌**可复用逻辑封装模式**‌，用于在多个组件之间共享有状态的业务逻辑。

------

### 什么是 Composables？

- ‌**本质**‌：一个返回响应式状态或方法的普通 JavaScript 函数。
- ‌**命名约定**‌：通常以 `use` 开头（如 `useCounter`、`useMousePosition`）。
- ‌核心能力‌：
  - 封装响应式状态（通过 `ref`、`reactive`）
  - 使用生命周期钩子（如 `onMounted`、`onUnmounted`）
  - 返回数据或方法供组件使用
- ‌**独立性**‌：每次调用 composables 都会创建新的实例，状态彼此隔离，除非主动共享外部状态。



## 各方案对比

| 方案                | 优点                         | 缺点               | 适用场景                |
| ------------------- | ---------------------------- | ------------------ | ----------------------- |
| **公共分享函数**    | 代码复用、统一管理、灵活配置 | 需要额外引入       | ✅ **推荐** 绝大多数场景 |
| 每个页面单独配置    | 独立性强、无依赖             | 代码重复、维护困难 | ❌ 不推荐，除非特殊需求  |
| 页面配置 + 公共函数 | 结合两者优点                 | 稍微复杂           | ✅ 大型项目              |
| 完全公共配置        | 最简单                       | 无法页面自定义     | ❌ 不灵活                |

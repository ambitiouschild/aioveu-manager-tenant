## ❌ 错误示例（你现在的问题根源）

```ts
else if (resData.code === ResultCodeEnum.TOKEN_INVALID) {
  setTimeout(() => {
    handleTokenExpiredSync();
  }, 0);
}
// ❌ 没有 reject → Promise 永远 pending
```

---

## ✅ 为什么必须加在这里？

| 原因             | 说明                     |
| ---------------- | ------------------------ |
| Promise 必须终结 | 否则调用方永远等不到     |
| reject ≠ 跳转    | 跳转是 UI，reject 是逻辑 |
| 防止业务继续执行 | 防止误判成功             |

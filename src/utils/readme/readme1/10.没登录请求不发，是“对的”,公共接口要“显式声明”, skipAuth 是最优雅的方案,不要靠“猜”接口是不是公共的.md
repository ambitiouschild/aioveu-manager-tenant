## 三、那「公共接口到底算什么？」

### ✅ 正确认知是：

| 接口类型    | 是否该带 token | 没登录时 |
| ----------- | -------------- | -------- |
| 登录 / 注册 | ❌              | ✅ 正常发 |
| 公共接口    | ❌ / ⚠️          | ✅ 正常发 |
| 业务接口    | ✅              | ❌ 被拦截 |

👉 问题不在 request
👉 问题在「你怎么定义公共接口」

---

## 四、✅ 正确解决方案（推荐）

### ✅ 方案一（最标准）：**skipAuth**

```ts
request({
  url: '/public/banner',
  method: 'GET',
  header: {
    skipAuth: true
  }
})
```

```ts
// 请求拦截器
if (config.header?.skipAuth === true) {
  return config; // ✅ 不处理 token
}
```

✅ 公共接口明确声明
✅ 不污染拦截器逻辑

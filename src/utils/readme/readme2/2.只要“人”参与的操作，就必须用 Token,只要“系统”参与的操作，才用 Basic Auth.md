## ✅ 为什么你这种情况必须用 Token？

因为你现在的操作是：

> “某个已登录的平台管理员，在操作 client”

| 要素     | 有没有 |
| -------- | ------ |
| 登录用户 | ✅ 有   |
| 用户身份 | ✅ 有   |
| 操作审计 | ✅ 需要 |
| 权限控制 | ✅ 需要 |

👉 这就是标准的“用户操作”，不是“系统行为”

## ✅ 你这个场景的正确架构（终局版）

```
平台管理员
 ↓
登录系统（用户名 + 密码）
 ↓
拿到 Token
 ↓
前端请求
Authorization: Bearer token
X-Client-Id: platform-admin
 ↓
Auth 校验 Token
 ↓
执行 启用 / 禁用
```

✅ Token 必须  
✅ X-Client-Id 可选（但强烈推荐）

---

## ✅ 那 Basic Auth 还用不用？

✅ 用，但只用于“系统级接口”

| 接口                | 用什么       |
| ------------------- | ------------ |
| 登录                | ❌ 不用 Basic |
| 启用 / 禁用 client  | ✅ Token      |
| 内部服务调用        | ✅ Basic Auth |
| 注册 client（系统） | ✅ Basic Auth |

## ✅ 你这个接口怎么写才对（终局版）

### ✅ Controller

```java
@PostMapping("/client/disable")
public Result<Void> disable(@RequestBody DisableClientDTO dto,
                            @RequestHeader("Authorization") String token) {

    // 1️⃣ 从 Token 解析当前登录用户
    String userId = resolveUserFromToken(token);

    // 2️⃣ 校验权限（平台管理员）
    checkPlatformAdmin(userId);

    // 3️⃣ 执行禁用
    clientService.disableClient(dto.getClientId());

    return Result.success();
}
```

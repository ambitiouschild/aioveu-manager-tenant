父组件**确实传递了** `goodsInfo`包含 `specList`和 `skuList`，但在 `loadGoodsData`中，你把它们复制到了本地变量 `specForm`和 `skuForm`，但**后续的修改没有同步回父组件**。

看看你的代码：

```
// 父组件传递的数据
const goodsInfo = computed({
  get: () => {
    return props.goodsInfo || {};
  },
  set: (value) => {
    emit("update:goodsInfo", value);
  }
});

// 加载时复制到本地变量
const loadGoodsData = async () => {
  // ...
  if (goodsInfo.value.specList?.length > 0) {
    await processSpecList(goodsInfo.value.specList); // 复制到 specForm
  }
  
  if (goodsInfo.value.skuList?.length > 0) {
    await processSkuList(goodsInfo.value.skuList); // 复制到 skuForm
  }
};
```

## 🎯 问题所在

1. **初始数据**：从 `goodsInfo`复制到 `specForm`/`skuForm`
2. **用户操作**：修改 `specForm`/`skuForm`
3. **丢失同步**：修改**没有**反映到 `goodsInfo`
4. **父组件**：看不到规格/库存的变化

## 🔧 解决方案

你需要**同步修改**回父组件。有两种方案：

### 方案一：每次修改都同步（推荐）

创建一个辅助函数，在修改规格/SKU时自动同步：

```
// 辅助函数：同步规格数据到父组件
const updateSpecForm = (updates: Partial<SpecFormData>) => {
  // 更新本地状态
  Object.assign(specForm.value, updates);
  
  // 同步到父组件
  const updated = {
    ...goodsInfo.value,
    specList: getFlatSpecList(), // 将嵌套结构扁平化
  };
  emit("update:goodsInfo", updated);
};

// 辅助函数：同步SKU数据到父组件
const updateSkuForm = (updates: Partial<SkuFormData>) => {
  // 更新本地状态
  Object.assign(skuForm.value, updates);
  
  // 同步到父组件
  const updated = {
    ...goodsInfo.value,
    skuList: skuForm.value.skuList.map(sku => ({
      ...sku,
      price: Math.round(Number(sku.price) * 100), // 元转分
    })),
  };
  emit("update:goodsInfo", updated);
};
```

### 方案二：修改所有操作函数

在每个修改函数中都添加同步：

```
// 添加规格
const handleAddSpec = () => {
  if (specForm.value.specList.length >= 3) {
    uni.showToast({ title: "最多支持3个规格", icon: "none" });
    return;
  }

  const newSpec = new Specification({
    id: generateTempId(),

    values: [],
  });

  specForm.value.specList.push(newSpec);
  specErrors.value.push("");

  // 🔴 新增：同步到父组件
  syncToParent();
  
  updateSpecTitles();
};

// 删除规格
const handleRemoveSpec = (index: number) => {
  // 验证逻辑...

  specErrors.value.splice(index, 1);

  // 🔴 新增：同步到父组件

};
```

### 方案三：在 watch 中自动同步（最简单）

```
// 监听规格变化，自动同步
watch(
  () => specForm.value,
  (newSpec) => {
    // 将嵌套结构转换为扁平结构
    const flatSpecList = getFlatSpecList();
    
    const updated = {
      ...goodsInfo.value,
      specList: flatSpecList,
    };
    emit("update:goodsInfo", updated);
  },
  { deep: true, immediate: true }
);

// 监听SKU变化，自动同步
watch(
  () => skuForm.value,
  (newSku) => {
    const updated = {
      ...goodsInfo.value,
      skuList: newSku.skuList.map(sku => ({
        ...sku,
        price: Math.round(Number(sku.price) * 100), // 元转分
      })),
    };
    emit("update:goodsInfo", updated);
  },
  { deep: true, immediate: true }
);

// 工具函数：将嵌套规格转换为扁平列表
const getFlatSpecList = () => {
  const result: any[] = [];
  specForm.value.specList.forEach(spec => {
    spec.values.forEach(value => {
      result.push({
        id: value.id,
        name: spec.name,
        value: value.value,
        picUrl: value.picUrl,
      });
    });
  });
  return result;
};
```

## 📋 完整同步方案

建议用**方案三**，在 `onMounted`中添加：

```
onMounted(() => {
  // 监听规格和SKU变化
  const stopSpecWatch = watch(
    () => specForm.value,
    (newSpec) => {
      const flatSpecList = getFlatSpecList();
      const updated = {
        ...goodsInfo.value,
        specList: flatSpecList,
      };
      emit("update:goodsInfo", updated);
    },
    { deep: true }
  );

  const stopSkuWatch = watch(
    () => skuForm.value,
    (newSku) => {
      const updated = {
        ...goodsInfo.value,
        skuList: newSku.skuList.map(sku => ({
          id: sku.id,
          skuSn: sku.skuSn,
          price: Math.round(Number(sku.price) * 100), // 元转分
          stock: sku.stock,
          Lockedstock: sku.Lockedstock,
          specIds: sku.specIds,
          specValues: sku.specValues,
          picUrl: sku.picUrl,
        })),
      };
      emit("update:goodsInfo", updated);
    },
    { deep: true }
  );

  onUnmounted(() => {
    stopSpecWatch();
    stopSkuWatch();
  });
});
```

## 💡 总结

你的问题确实存在：

1. 数据从父组件 `goodsInfo`复制到本地 `specForm`/`skuForm`
2. 用户操作修改了本地变量
3. 但**没有同步回父组件**
4. 父组件看到的还是**旧数据**

用 **watch 监听 + 自动同步** 是解决这个问题最简洁的方式。这样用户在界面上做的任何修改，都会实时同步到父组件。

| 需求             | 你的方案                                   | 为什么合理                       |
| ---------------- | ------------------------------------------ | -------------------------------- |
| **多步骤表单**   | GoodsInfo → 实时同步 → 父组件 → GoodsStock | 步骤间数据传递必须通过父组件     |
| **步骤可跳转**   | 父组件集中管理，支持任意步骤切换           | 如果直接传递，回退步骤就丢失数据 |

| **数据量差异**   | Info(小)实时同步 vs Stock(大)本地管理      | 按数据特性选择最优策略           |

## 🎯 架构优势


### 3. **用户体验好**

- 可任意切换步骤不丢数据
- 实时保存基本信息（防丢失）
- 复杂操作（规格）有本地草稿

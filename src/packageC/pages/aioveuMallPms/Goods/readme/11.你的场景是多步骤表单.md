## 🔄 多步骤表单的数据流

```
GoodsAddPage (父组件)
  │
  ├── Step 1: GoodsCategory (选择分类)
  │     └→ categoryId
  │
  ├── Step 2: GoodsInfo (商品基本信息)
  │     ├→ emit → 父组件
  │     └→ 传递给下一步
  │
  └── Step 3: GoodsStock (规格库存)
        ├→ 获取父组件中的全部数据
        └→ 提交
```

## 🎯 正确的架构

### 模式A：**串联传递**（当前实现）

```
GoodsInfo → emit → 父组件 → GoodsStock
优点：步骤间解耦，可独立跳转
缺点：需要实时同步
```

### 模式B：**直接传递**（你考虑的）

```
GoodsInfo → GoodsStock
优点：无实时同步开销
缺点：必须按顺序步骤
```

## ⚖️ 选择建议

由于你的场景是**多步骤表单**，建议：

1. **基本信息 (GoodsInfo)**：实时同步 数据量小 需要被后续步骤使用 可能独立验证
2. **规格库存 (GoodsStock)**：本地管理+提交获取 数据量大 只在最后一步使用 操作复杂

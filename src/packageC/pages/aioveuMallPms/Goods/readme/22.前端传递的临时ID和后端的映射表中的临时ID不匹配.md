## 🔍 问题分析

从日志可以看到：

1. **前端传递的SKU specIds**： `specIds: temp_1778161675312_jn9m387pj|temp_1778161683800_04psidio7`
2. **后端规格值映射表**： `映射表内容: {temp_1778161692321_p7g0vsp4v=961, temp_1778161692321_9dv3613rr=962}`
3. **问题**：前端传递的临时ID和后端的映射表中的临时ID**不匹配**！ 前端：`temp_1778161675312_jn9m387pj`和 `temp_1778161683800_04psidio7` 后端映射表：`temp_1778161692321_p7g0vsp4v`和 `temp_1778161692321_9dv3613rr`
4. **结果**：映射查找返回 `null`，拼接成 `null_null`

## ✅ 解决方案

### 问题根源

规格值的临时ID在**前端生成**和后端**保存时生成**的不一致。您需要确保：

1. 前端传递的临时ID
2. 后端保存规格值时生成的临时ID
3. 后端映射表中查找的临时ID

**这三者必须一致**。

### 方案1：统一临时ID生成逻辑（推荐）

**在前端生成规格值时，保存这个临时ID，并在保存规格值时传递给后端**。

修改前端代码：

```
// 在 handleInputSpecValue 中
```

### 方案2：在后端重新建立映射（如果前端已传递临时ID）

```
        log.info("⚠️ 生成临时ID并建立映射: {} -> {}", tempId, savedValue.getId());
    }
}
```

**根本问题**：前端传递的临时ID和后端建立映射的临时ID不一致。

1. ✅ 确保前端传递临时ID

## 问题所在

1. **编辑模式**：当商品已经有 SKU 数据时，`processSkuList`会处理并显示**已有的SKU数据**
2. **重新生成**：`generateSkuList()`会基于当前规格重新计算笛卡尔积，生成**全新的SKU列表**
3. **数据覆盖**：新生成的 SKU 会覆盖已有的 SKU 数据，导致编辑模式下的数据丢失

## 正确的逻辑

### 情况1：编辑模式（商品有ID）

- 只处理已有的规格和SKU数据
- **不重新生成**SKU列表
- 保持原有的SKU数据不变

### 情况2：新增模式（商品无ID）

- 如果有规格数据，可以生成SKU列表
- 如果还没有规格数据，等待用户配置

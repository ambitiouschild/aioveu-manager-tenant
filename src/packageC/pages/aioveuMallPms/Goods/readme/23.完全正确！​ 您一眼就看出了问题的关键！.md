2. SKU的specIds引用这个ID：`temp_123456789_abc|temp_987654321_def`

4. 后端找不到匹配：`temp_999999999_xyz` 不在映射表中

## 🎯 总结

问题：临时ID在提交时被重新生成，但SKU引用的是旧的临时ID。

解决方案：提交规格值时，不要重新生成临时ID，使用规格值对象中已有的ID。

这样就能保证：
1. SKU的`specIds`引用的是`temp_123`
2. 提交的规格值ID也是`temp_123`
3. 后端能正确映射，不会产生`null_null`

您遇到了iOS/Android 键盘数字输入限制问题。这是 `type="digit"` 的一个已知问题。

## 问题原因

在 uni-app 中，`type="digit"` 在某些平台（特别是 iOS）上有特殊行为：
1. 输入框只允许输入数字和小数点
2. 当已有值时，键盘的"完成"按钮可能会清空输入
3. 不能像普通文本那样在中间插入编辑

## 最简单的解决方案

### 方案1：改为 `type="number"`（最推荐）

```vue
<!-- 改为 type="number" -->
<input
  v-model="goodsInfo.price"
  placeholder="请输入促销价"
  type="number"  <!-- 改为 number -->
  class="price-input"
  placeholder-class="placeholder"
  @blur="formatPrice('price')"
/>
```

`type="number"` 允许：
• 在任意位置插入数字

• 调起数字键盘

• 更自然的编辑体验


### 方案2：改为 `type="text"` 并添加输入限制

```vue
<!-- 改为 type="text"，手动控制输入 -->
<input
  v-model="priceText"
  placeholder="请输入促销价"
  type="text"  <!-- 改为 text -->
  class="price-input"
  placeholder-class="placeholder"
  @input="handlePriceTextInput('price')"
  @blur="applyPriceText('price')"
/>
```

```javascript
// 文本形式的临时变量
const priceText = ref('');
const originPriceText = ref('');

// 输入时控制只能输入数字和小数点
const handlePriceTextInput = (event, field) => {
  const value = event.detail?.value || '';
  let filtered = value.replace(/[^\d.]/g, '');
  
  // 确保只有一个点
  const parts = filtered.split('.');
  if (parts.length > 2) {
    filtered = parts[0] + '.' + parts.slice(1).join('');
  }
  
  // 限制小数位数
  if (parts.length > 1) {
    filtered = parts[0] + '.' + parts[1].substring(0, 2);
  }
  
  if (field === 'price') {
    priceText.value = filtered;
  } else {
    originPriceText.value = filtered;
  }
};
```

## 推荐方案

改为 `type="number"`，这是改动最小、效果最好的：

```vue
<!-- 零售价 -->
<input
  v-model="goodsInfo.originPrice"
  placeholder="请输入零售价"
  type="number"  <!-- 只改这里 -->
  class="price-input"
  placeholder-class="placeholder"
  @blur="formatPrice('originPrice')"
/>

<!-- 促销价 -->
<input
  v-model="goodsInfo.price"
  placeholder="请输入促销价"
  type="number"  <!-- 只改这里 -->
  class="price-input"
  placeholder-class="placeholder"
  @blur="formatPrice('price')"
/>
```

```javascript
// 格式化函数保持不变
const formatPrice = (field) => {
  const value = goodsInfo.value[field];
  if (value === '' || value === null || value === undefined) {
    goodsInfo.value[field] = 0;
  } else {
    const num = parseFloat(value);
    goodsInfo.value[field] = isNaN(num) ? 0 : parseFloat(num.toFixed(2));
  }
  validateField(field);
};
```

## 为什么 `type="number"` 更好？

1. 编辑自由：可以在任意位置插入、删除数字
2. 数字键盘：在移动端仍会调起数字键盘
3. 跨平台：在 iOS 和 Android 上行为一致
4. 简单：只需要改一个属性

## 总结

最小改动：将 `type="digit"` 改为 `type="number"`

这样用户就可以：
1. 在已有数字中间插入新数字
2. 正常编辑，不需要全删
3. 在移动端使用数字键盘
4. 保持原有的格式化功能

看到这个错误，**核心问题是：无限递归更新循环**。这是因为在 `computed`的 setter 中触发了更新，而更新又触发了 computed 的 getter，形成了死循环。

## 问题分析

从错误堆栈可以看出：

1. **`Maximum recursive updates exceeded`** - 递归更新超过限制
2. **`resetAttributeList`→ `set`→ `handleGoodsInfoUpdate`** - 更新链
3. **`calculateTableHeight`中的 `in(this)`出错** - 使用了错误的 this 上下文

## 核心问题位置

### 1. 商品信息的 computed 导致递归

```
// ❌ 问题代码
const goodsInfo = computed({
  get: (): GoodsInfoData => {
    const data = props.goodsInfo || {};
    if (!data.attrList || !Array.isArray(data.attrList)) {
      data.attrList = [new GoodsAttribute()];  // ❌ 这里修改了原始数据
    }
    return {
      id: data.id,
      categoryId: data.categoryId,
      attrList: data.attrList.map((attr) => new GoodsAttribute(attr)),
    };
  },
  set: (value: GoodsInfoData) => {
    emit("update:goodsInfo", value);  // ❌ 触发父组件更新
  },
});
```

### 2. `resetAttributeList`函数

```
const resetAttributeList = (): void => {
  const newGoodsInfo = { ...goodsInfo.value, attrList: [new GoodsAttribute()] };
  goodsInfo.value = newGoodsInfo;  // ❌ 触发 computed 的 setter
  errors.value = [{}];
  console.log("🔄 重置属性列表");
};
```

### 3. `calculateTableHeight`函数中的 this 问题

```
const calculateTableHeight = () => {
  const query = uni.createSelectorQuery().in(this);  // ❌ this 在 setup 中不是组件实例
  // ...
};
```

## 修复方案

### 方案1：修复计算属性（最重要）

```
// 修复商品信息 computed
const goodsInfo = computed({
  get: (): GoodsInfoData => {
    const data = props.goodsInfo || {};
    
    // ❌ 不要修改原始数据
    // ✅ 返回新对象
    return {
      id: data.id,
      categoryId: data.categoryId,
      // 确保 attrList 始终是数组
      attrList: Array.isArray(data.attrList) 
        ? data.attrList.map(attr => new GoodsAttribute(attr))
        : [new GoodsAttribute()],  // 返回新数组，不修改原始数据
    };
  },
  set: (value: GoodsInfoData) => {
    // 添加防重复触发检查
    const current = props.goodsInfo;
    if (JSON.stringify(current) === JSON.stringify(value)) {
      console.log("跳过相同数据的更新");
      return;
    }
    console.log("触发商品信息更新", value);
    emit("update:goodsInfo", value);
  },
});
```

// 修复 watchCategoryChange
const setupCategoryWatch = () => {
  let isHandlingCategoryChange = false;
  
  watch(
    () => goodsInfo.value.categoryId,

          await loadCategoryAttributes(newCategoryId);
        } else {
          // 没有选择分类，清空属性
          resetAttributeList();
        }
      } finally {
        isHandlingCategoryChange = false;
      }
    },
    {

      deep: false,
    }

## 完整修复代码

```
<script setup lang="ts">
import { ref, computed, watch, onMounted, onUnmounted, nextTick, getCurrentInstance } from "vue";

// 获取组件实例
const instance = getCurrentInstance();

// 使用本地 ref 存储属性列表
const localAttrList = ref<GoodsAttributeData[]>([]);
const errors = ref<AttributeError[]>([]);
const tableHeight = ref<number>(300);
const loading = ref<boolean>(false);
const systemInfo = uni.getSystemInfoSync();

// 商品信息 computed - 简化版
const goodsInfo = computed(() => {
  const data = props.goodsInfo || {};
  return {
    id: data.id,
    categoryId: data.categoryId,
    attrList: localAttrList.value, // 使用本地 ref
  };
});

// 从 props 同步数据
const syncAttrListFromProps = () => {
  const data = props.goodsInfo || {};
  const attrList = Array.isArray(data.attrList) 
    ? data.attrList.map(attr => new GoodsAttribute(attr))
    : [new GoodsAttribute()];
  
  localAttrList.value = attrList;
  errors.value = attrList.map(() => ({}));
};

// 更新属性列表
const updateAttributeList = (newList: GoodsAttributeData[]) => {
  localAttrList.value = newList;
  
  // 触发更新事件
  const updatedGoodsInfo = {
    ...props.goodsInfo,
    attrList: newList
  };
  emit("update:goodsInfo", updatedGoodsInfo);
};

// 重置属性列表
const resetAttributeList = (): void => {
  const newList = [new GoodsAttribute()];
  updateAttributeList(newList);
  console.log("🔄 重置属性列表");
};

// 计算表格高度
const calculateTableHeight = () => {
  if (!instance) return;
  
  nextTick(() => {
    const query = uni.createSelectorQuery().in(instance);
    query
      .select(".component-container")
      .boundingClientRect((data: any) => {
        if (data) {
          const windowHeight = systemInfo.windowHeight;
          const containerTop = data.top;
          tableHeight.value = windowHeight - containerTop - 220; // 简化计算
        }
      })
      .exec();
  });
};

// 处理属性输入
const handleAttributeInput = (event: any, index: number, field: "name" | "value"): void => {
  const value = event?.detail?.value || '';
  
  if (index < 0 || index >= localAttrList.value.length) {
    console.error("索引越界");
    return;
  }
  
  const newList = [...localAttrList.value];
  newList[index] = { ...newList[index], [field]: value };
  updateAttributeList(newList);
  
  // 清除错误
  if (errors.value[index]) {
    const newErrors = [...errors.value];
    delete newErrors[index][field];
    errors.value = newErrors;
  }
};

// 添加属性
const handleAddAttribute = (): void => {
  const newList = [...localAttrList.value, new GoodsAttribute()];
  updateAttributeList(newList);
  errors.value.push({});
};

// 删除属性
const handleRemoveAttribute = (index: number): void => {
  if (localAttrList.value.length <= 1) {
    uni.showToast({ title: "至少需要保留一个属性", icon: "none" });
    return;
  }
  
  const newList = localAttrList.value.filter((_, i) => i !== index);
  updateAttributeList(newList);
  errors.value.splice(index, 1);
};

// 生命周期
onMounted(() => {
  console.log("🔄 商品属性组件挂载");
  syncAttrListFromProps();
  calculateTableHeight();
});

// 监听 props 变化
watch(() => props.goodsInfo, () => {
  syncAttrListFromProps();
}, { deep: true });

// 暴露方法
defineExpose({
  validateForm: async () => {
    return validateAllFields() && checkDuplicateNames();
  },
  getAttributes: () => localAttrList.value,
  clearAttributes: resetAttributeList,
});
</script>
```

## 关键修复点

1. **使用本地 ref 存储数据**，避免 computed 递归
2. **移除 computed 中的 setter 递归**
3. **修复 calculateTableHeight 中的 this 问题**
4. **避免 watch 的 immediate: true**
5. **添加防重复更新检查**

这样应该能解决递归更新和 this 上下文的问题。

你说到点子上了！这正是**组件设计模式的选择问题**。

## 🤔 两种模式的对比

### 模式A：实时同步（当前 GoodsInfo）

```
// GoodsInfo.vue
// 每次修改都 emit
const handleNameInput = (e) => {
  const updated = { ...goodsInfo.value, name: e.detail.value };
  emit("update:goodsInfo", updated);  // 实时同步
}
```

### 模式B：提交时获取（当前 GoodsStock）

```
// GoodsStock.vue
// 本地管理，提交时获取
const handleSubmit = async () => {
  const allData = {
    ...goodsInfo.value,               // 基本信息
    specList: getSpecListData(),       // 从组件内获取
    skuList: getSkuListData()         // 从组件内获取
  };
  // 提交
}
```

## 🔄 为什么两个组件策略不同？

这是**由组件特性决定的**：

| 组件           | 数据量        | 操作频率      | 复杂度 | 推荐模式     |
| -------------- | ------------- | ------------- | ------ | ------------ |
| **GoodsInfo**  | 小 (基本信息) | 低 (填表单)   | 低     | 实时同步 ✅   |
| **GoodsStock** | 大 (规格/SKU) | 高 (频繁操作) | 高     | 提交时获取 ✅ |

## 🎯 GoodsInfo 可以改成 GoodsStock 模式吗？

**完全可以！** 而且这样做**更一致、更简单**。

### 修改方案：

```
// GoodsInfo.vue - 改为本地管理
const localGoodsInfo = ref({});

// 加载时从 props 复制
onMounted(() => {
  localGoodsInfo.value = { ...props.goodsInfo };
});

// 所有输入函数直接修改本地变量
const handleNameInput = (e) => {
  localGoodsInfo.value.name = e.detail.value;  // 直接改，不 emit
};
```

### 父组件需要修改：

```
// GoodsAddPage.vue
// 提交时需要获取 GoodsInfo 的数据
const handleSubmit = async () => {
  // 1. 获取 GoodsInfo 的数据
  const goodsInfoData = await getGoodsInfoData(); // 需要暴露方法
  
  // 2. 获取 GoodsStock 的数据
  const goodsStockData = await getGoodsStockData(); // 需要暴露方法
  
  // 3. 合并提交
  const allData = {
    ...goodsInfoData,
    ...goodsStockData
  };
  
  // 提交...
};
```

## 🔧 如何暴露组件数据？

在子组件中暴露方法：

```
// GoodsInfo.vue
const getData = () => {
  return { ...localGoodsInfo.value };
};

defineExpose({ getData });
```

```
// GoodsStock.vue
const getData = () => {
  return {
    specList: getFlatSpecList(),
    skuList: skuForm.value.skuList
  };
};

defineExpose({ getData });
```

父组件调用：

```
// GoodsAddPage.vue
const goodsInfoRef = ref();
const goodsStockRef = ref();

const handleSubmit = async () => {
  const goodsInfoData = goodsInfoRef.value?.getData() || {};
  const goodsStockData = goodsStockRef.value?.getData() || {};
  
  const allData = { ...goodsInfoData, ...goodsStockData };
  // 提交...
};
```

## 🆚 两种策略对比

| 方面   | 实时同步               | 提交时获取      |
| ------ | ---------------------- | --------------- |
| 复杂度 | 高 (每个修改都要 emit) | 低 (组件内管理) |
| 性能   | 差 (频繁 emit)         | 好 (无额外开销) |
| 代码量 | 多 (emit + watch)      | 少 (直接修改)   |
| 调试   | 难 (数据流复杂)        | 易 (状态清晰)   |
| 一致性 | 与父组件强耦合         | 组件自治        |

## ✅ 建议

**强烈建议统一为"提交时获取"模式**，因为：

1. **架构一致**：两个组件都用相同模式
2. **代码简化**：去掉大量 emit/watch
3. **性能更好**：减少不必要的通信
4. **维护方便**：组件内聚，职责清晰

## 🔄 修改步骤

1. 在 GoodsInfo 中改为本地 `ref`
2. 移除所有 `emit("update:goodsInfo")`
3. 暴露 `getData()`方法
4. 父组件在提交时调用 `getData()`获取
5. GoodsStock 保持现有模式

这样两个组件的架构就完全一致了，代码更清晰！

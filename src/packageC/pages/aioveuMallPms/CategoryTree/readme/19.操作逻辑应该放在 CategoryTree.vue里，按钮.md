这些操作，放在category-tree里还时flat-tree里

**操作逻辑应该放在 `CategoryTree.vue`里，按钮应该放在 `flat-tree.vue`里。**

## 📁 正确的职责划分

### 1. **`flat-tree.vue`** (只负责UI展示)

- 显示树形结构
- 显示**操作按钮**（编辑、新增下级、删除）
- 点击按钮时**只触发事件**，不处理业务逻辑

```
<!-- flat-tree.vue 模板部分 -->
<view class="node-actions">
  <text class="action-btn edit" @tap.stop="$emit('edit-click', item)">编辑</text>
  <text class="action-btn add" @tap.stop="$emit('add-click', item)">+ 新增下级</text>
  <text class="action-btn delete" @tap.stop="$emit('delete-click', item)">删除</text>
</view>
```

### 2. **`CategoryTree.vue`** (负责业务逻辑)

- 处理**所有业务逻辑**（API调用、数据管理、弹窗控制）
- 监听 `flat-tree`的事件
- 控制弹窗显示/隐藏
- 提交表单数据

```
// CategoryTree.vue
<FlatTree
  :data="treeData"
  @edit-click="onUpdate"      <!-- 监听编辑事件 -->
  @add-click="onAdd"         <!-- 监听新增下级事件 -->
  @delete-click="onDelete"   <!-- 监听删除事件 -->
/>
```

## ✅ 最清晰的架构

### 文件结构：

```
CategoryTree/
├── CategoryTree.vue          # 主文件，包含所有业务逻辑
│   ├── 弹窗控制
│   ├── 表单处理  
│   ├── API调用
│   └── 数据管理
└── components/
    └── flat-tree.vue         # 纯UI组件，只负责显示
        ├── 树形渲染
        ├── 操作按钮
        └── 触发事件
```

### 数据流向：

```
用户点击"编辑"按钮
↓
flat-tree.vue 触发 @edit-click 事件
↓
CategoryTree.vue 接收事件，执行 onUpdate 方法
↓
弹出编辑对话框，填充表单数据
↓
用户提交 → 调用API → 更新数据
```

## 🔧 如果觉得按钮太复杂，可以简化

### 方案一：右键菜单（推荐）

```
<!-- flat-tree.vue -->
<view 
  class="node-content"
  @longpress="showContextMenu(item)"
  @tap="handleClick(item)"
>
  <text>{{ item.name }}</text>
</view>

<!-- 长按菜单 -->
<view v-if="showMenu" class="context-menu">
  <view @click="edit">编辑</view>
  <view @click="addChild">新增下级</view>
  <view @click="deleteNode">删除</view>
</view>
```

### 方案二：悬浮按钮

```
<!-- flat-tree.vue -->
<view 
  class="node-content"
  @mouseenter="showActions = true"
  @mouseleave="showActions = false"
>
  <text>{{ item.name }}</text>
  <view v-if="showActions" class="hover-actions">
    <text @click="$emit('edit-click', item)">编辑</text>
    <text @click="$emit('delete-click', item)">删除</text>
  </view>
</view>
```

### 方案三：简洁按钮（只保留图标）

```
<!-- flat-tree.vue -->
<view class="node-actions">
  <text class="icon-btn" @click="$emit('edit-click', item)">✏️</text>
  <text class="icon-btn" @click="$emit('add-click', item)">➕</text>
  <text class="icon-btn" @click="$emit('delete-click', item)">🗑️</text>
</view>
```

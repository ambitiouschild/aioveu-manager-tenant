这意味着：`<CategoryTreeNode>`递归组件**没有执行 `v-for`循环**。

## 🎯 **问题根源**

在 `CategoryTreeNode.vue`中，子节点渲染条件是：

```
<view v-if="expanded && node.children && node.children.length > 0" class="children-container">
```

**但这里有个关键细节**：`expanded`的计算是基于 `props.expandedNodes.has(props.node.id)`。

**问题**：`expandedNodes`在递归传递时，**子组件可能接收到了不同的值**！

## 🎯 **最可能的问题**

**`expandedNodes`在递归传递时类型发生了变化**。

在 Vue 3 中，`Set`对象**不是深度响应式**的。当传递给子组件时，可能失去了 `Set`的方法。

**解决方案**：改为使用**响应式 Set** 或使用数组。

在 `CategoryTree.vue`中：

```
import { reactive } from 'vue'

// 使用 reactive 包装 Set
const expandedNodes = reactive({
  value: new Set([0]),
  has(id) {
    return this.value.has(id)
  },
  add(id) {
    this.value.add(id)
  },
  delete(id) {
    this.value.delete(id)
  }
})
```

或者在 `CategoryTreeNode.vue`中改为使用数组：

```
// 接收数组
const props = defineProps<{
  expandedNodes: number[]  // 改为数组
}>()

// 计算展开状态
const expanded = computed(() => {
  return props.expandedNodes.includes(props.node.id)
})
```

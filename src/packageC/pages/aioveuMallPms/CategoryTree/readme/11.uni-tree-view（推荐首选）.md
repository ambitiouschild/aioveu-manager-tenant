既然自研递归卡住了，直接引入成熟组件是最快方案。针对你的**商品分类树**场景，推荐使用 **uni-tree-view**（功能全、上手快）或 **zx-tree**（大数据量、高性能）。

## 🚀 方案一：uni-tree-view（推荐首选）

**适用场景**：标准分类树，节点数在几百以内，需要快速跑通业务。

### 1. 安装引入

```
npm install uni-tree-view
```

### 2. 页面使用（直接替换你的 CategoryTree.vue）

```
<!-- CategoryTree.vue -->
<template>
  <view class="category-tree">
    <uni-tree-view
      :data="categoryOptions"
      :treeProps="{ id: 'id', label: 'name', children: 'children' }"
      @node-click="onNodeClick"
    />
  </view>
</template>

<script setup lang="ts">
import { ref } from 'vue';
import UniTreeView from 'uni-tree-view';

// 你的分类数据（确保是嵌套 children 结构）
const categoryOptions = ref([
  {
    id: 1,
    name: '一级分类',
    children: [
      { id: 11, name: '二级分类', children: [{ id: 111, name: '三级分类' }] }
    ]
  }
]);

const onNodeClick = (node) => {
  console.log('点击节点:', node);
  // 这里可以触发你的编辑/删除逻辑
};
</script>

<style scoped>
.category-tree {
  height: 100vh;
  background: #fff;
}
</style>
```



### 3. 数据适配（关键！）

如果后端返回的是**扁平数组**（带 parentId），需要先转成**嵌套 children 结构**：

```
// utils/tree.js
export function buildTree(flatData, parentId = null) {
  return flatData
    .filter(item => item.parentId === parentId)
    .map(item => ({
      ...item,
      children: buildTree(flatData, item.id)
    }));
}

// 在页面中调用
import { buildTree } from '@/utils/tree.js';
// 假设 list 是接口返回的扁平数组
categoryOptions.value = buildTree(list);
```

## 🛠️ 操作按钮集成（编辑/删除）

第三方组件通常不支持直接插槽，建议采用**长按菜单**或**外部工具栏**：

```
<template>
  <view>
    <uni-tree-view
      :data="categoryOptions"
      @node-click="onNodeClick"
      @node-longpress="onNodeLongPress" <!-- 长按触发操作菜单 -->
    />
    
    <!-- 操作弹窗 -->
    <uni-popup ref="popup">
      <view class="action-sheet">
        <button @click="onEdit">编辑</button>
        <button @click="onDelete">删除</button>
      </view>
    </uni-popup>
  </view>
</template>

<script setup>
const currentNode = ref(null);

const onNodeLongPress = (node) => {
  currentNode.value = node;
  // 显示底部操作菜单
  uni.showActionSheet({
    itemList: ['编辑', '删除'],
    success: (res) => {
      if (res.tapIndex === 0) onEdit(node);
      else if (res.tapIndex === 1) onDelete(node);
    }
  });
};
</script>
```

## ⚠️ 避坑指南

1. **数据格式**：99% 的渲染问题都是数据格式不对，必须确保是 `[{ id, name, children: [...] }]`嵌套结构。
2. **组件名注册**：如果报错找不到组件，检查 `pages.json`是否配置了 `"style": "v3"`（Vue 3 模式）。
3. **样式穿透**：如果需要修改节点样式，使用 `::v-deep`深度选择器。

**建议**：先按 **方案一（uni-tree-view）** 跑通，半小时内就能看到效果，别再死磕递归了。

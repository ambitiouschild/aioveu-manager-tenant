问题在于**事件冒泡冲突**。你给整行添加了 `@tap="handleItemClick(item)`，但同时也给内部的按钮加了 `@tap.stop`，导致点击按钮时阻止了整行事件。

## 🔧 最小修复方案

在 `flat-tree.vue`中修改，只改**点击事件绑定顺序**：

### 1. 修改整行点击逻辑

```
<!-- 修改 flat-tree.vue 的模板 -->
<view
  v-for="item in displayList"
  :key="item.id"
  class="tree-item"
  :style="{
    paddingLeft: (item.level * 40) + 'rpx',
    display: item._visible ? 'flex' : 'none'
  }"
  @tap.stop="handleItemClick(item)"  <!-- 添加 stop 防止冒泡到父级 -->
>
```

### 2. 修改 `handleItemClick`方法

```
/**
 * 整行点击事件
 */
const handleItemClick = (item: FlatTreeNode) => {
  console.log('点击整行:', item.name)
  
  // 如果点击的是展开图标区域，不处理（由 expand-icon 的 @tap.stop 处理）
  // 判断是否点击了展开图标区域
  const target = event?.target as HTMLElement
  if (target && target.closest('.expand-icon')) {
    console.log('点击了展开图标，跳过整行点击')
    return
  }
  
  // 如果点击了操作按钮区域，不处理
  if (target && target.closest('.node-actions')) {
    console.log('点击了操作按钮，跳过整行点击')
    return
  }
  
  // 否则，如果节点有子节点，切换展开
  if (item.children && item.children.length > 0) {
    console.log('切换节点展开状态')
    toggleExpand(item)
  } else {
    // 没有子节点，触发正常的节点点击
    handleClick(item)
  }
}
```

### 3. 或者更简单：修改事件绑定顺序

直接在模板中调整，让**操作按钮的事件最后执行**：

```
<view
  v-for="item in displayList"
  :key="item.id"
  class="tree-item"
  :style="{
    paddingLeft: (item.level * 40) + 'rpx',
    display: item._visible ? 'flex' : 'none'
  }"
  @tap="handleRowClick(item)"  <!-- 改成 handleRowClick -->
>
```

添加新方法：

```
/**
 * 行点击事件
 */
const handleRowClick = (item: FlatTreeNode) => {
  console.log('行点击:', item.name)
  
  // 延迟执行，让按钮点击事件先处理
  setTimeout(() => {
    // 如果点击了按钮，不展开
    if (item.children && item.children.length > 0) {
      toggleExpand(item)
    } else {
      handleClick(item)
    }
  }, 10)
}
```

## ⚡ 最简单的解决方案

实际上，**你已经有展开图标了**，可以直接用展开图标。但如果非要点击整行展开，最简单的方案是：

### 方案A：只在没有按钮的区域点击才展开

```
/**
 * 行点击事件
 */
const handleRowClick = (item: FlatTreeNode, event: any) => {
  // 获取点击位置
  const clientX = event?.touches?.[0]?.clientX || 0
  const clientY = event?.touches?.[0]?.clientY || 0
  
  // 计算按钮区域
  const buttonWidth = 120 // 按钮区域大概宽度
  const screenWidth = uni.getSystemInfoSync().windowWidth
  
  // 如果点击位置在按钮区域（屏幕右侧120px内），不展开
  if (clientX > screenWidth - buttonWidth) {
    console.log('点击了按钮区域')
    return
  }
  
  // 否则切换展开
  if (item.children && item.children.length > 0) {
    toggleExpand(item)
  } else {
    handleClick(item)
  }
}
```

### 方案B：用CSS区分点击区域（推荐）

```
<view class="tree-item">

  </view>
</view>
```

## 🎯 我的建议

用**方案B**最简单明了，把界面分成两部分：

- **左侧名称区**：点击切换展开/收起
- **右侧按钮区**：点击触发操作

这样逻辑清晰，不会冲突。

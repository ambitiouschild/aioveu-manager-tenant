明白了！你需要**固定三级分类**，这比无限层级更简单，性能更好，也更符合业务需求。

## ✅ 三级分类架构

```
全部分类 (level 0)
├── 一级分类 (level 1)  ← 大家电、小家电等
│   ├── 二级分类 (level 2) ← 厨房电器、清洁电器等
│   │   └── 三级分类 (level 3) ← 冰箱、烤箱等
```

## 🔧 修改方案

### 1. 在 `CategoryTree.vue`中限制层级

在 `onAdd`方法中添加层级限制：

```
/**
 * 新增分类
 */
const onAdd = (node: CategoryNode) => {
  dialogTitle.value = node.id === 0 ? '新增一级分类' : '新增下级分类'
  popupType.value = 'info'
  resetForm()
  formData.id = undefined

  if (node.id === 0) {
    // 新增一级分类
    parentCategory.id = 0
    parentCategory.name = '全部分类'
    parentCategory.level = 0
    formData.parentId = 0
    formData.level = 1
  } else {
    // 新增下级分类
    parentCategory.id = node.id
    parentCategory.name = node.name
    parentCategory.level = node.level
    
    // ✅ 关键：限制最多三级
    if (node.level >= 3) {
      uni.showToast({
        title: '最多只能创建三级分类',
        icon: 'none'
      })
      return
    }
    
    formData.parentId = node.id
    formData.level = (node.level || 0) + 1
  }

  popupRef.value?.open()
}
```

## 🎯 业务规则定义

| 当前节点           | 可新增子级 | 说明                         |
| ------------------ | ---------- | ---------------------------- |
| 全部分类 (level 0) | ✅ 一级分类 | 最多创建N个一级分类          |
| 一级分类 (level 1) | ✅ 二级分类 | 每个一级分类下可创建二级分类 |
| 二级分类 (level 2) | ✅ 三级分类 | 每个二级分类下可创建三级分类 |
| 三级分类 (level 3) | ❌ 不能新增 | 三级分类是最终级别           |

## 📊 数据验证

在 `buildTree`函数中也可以添加验证：

```
function buildTree(list: CategoryNode[], parentId: number = 0, level: number = 0): CategoryNode[] {
  // 限制最大层级
  if (level >= 3) {
    console.warn('达到最大层级限制')
    return []
  }
  
  return list
    .filter(item => item.parentId === parentId)
    .map(item => {
      const children = buildTree(list, item.id, level + 1)
      return {
        ...item,
        level: level,
        expanded: false,
        children: children
      }
    })
}
```

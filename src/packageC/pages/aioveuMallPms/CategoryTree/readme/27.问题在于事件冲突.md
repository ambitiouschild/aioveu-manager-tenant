问题在于**事件冲突**。你给整行加了 `@tap.stop`，但同时也给内部的 `expand-icon`和 `node-content`加了 `@tap.stop`，导致事件被多次阻止。

## 🔧 最小修复方案

### 1. 移除 `node-content`的点击事件

因为整行已经处理点击了，`node-content`的点击事件是多余的：

```
<!-- 找到这行 -->
<view
  class="node-content"
  :class="{ selected: item.id === selectedId }"
  @tap.stop="handleClick(item)"  <!-- ❌ 删除这行 -->
>
```

改为：

```
<view
  class="node-content"
  :class="{ selected: item.id === selectedId }"
>
```

### 2. 修改 `handleRowClick`方法

确保它正确处理叶子节点：

```
/**
 * ✅ 修复：点击整行展开/收起
 */
const handleRowClick = (item: FlatTreeNode) => {
  console.log('点击行:', item.name)
  
  // 如果有子节点，切换展开状态
  if (item.children && item.children.length > 0) {
    console.log('有子节点，切换展开状态')
    toggleExpand(item)
  } else {
    // 叶子节点，触发点击事件
    console.log('叶子节点，触发点击')
    selectedId.value = item.id
    emit('nodeClick', item)
  }
}
```

### 3. 简化 `handleClick`方法

因为现在点击整行就会触发，所以 `handleClick`可以简化或移除。

## 🔄 完整修复

如果上述都无效，用这个**终极方案**：

```
const handleRowClick = (item: FlatTreeNode, event: any) => {
  // 阻止默认行为
  event.preventDefault()
  event.stopPropagation()
  
  console.log('点击了:', item.name)
  
  // 如果点击了按钮区域，不处理
  if (event.target.closest('.node-actions') || 
      event.target.closest('.action-btn')) {
    console.log('点击了按钮，跳过')
    return
  }
  
  if (item.children && item.children.length > 0) {
    // 直接修改展开状态
    const index = flatNodes.value.findIndex(n => n.id === item.id)
    if (index !== -1) {
      flatNodes.value[index].expanded = !flatNodes.value[index].expanded
      flatNodes.value = [...flatNodes.value]  // 强制更新
    }
  } else {
    selectedId.value = item.id
    emit('nodeClick', item)
  }
}
```
